# 水産物貿易 書類確認・申請準備支援 AI Agent
# ユーザーガイド（利用者向け完全マニュアル）

**バージョン:** 1.0（MVP）
**対象読者:** 水産物輸入会社の実務担当者・貿易事務担当者・案件承認者
**前提知識:** 貿易書類（Invoice、Packing List、B/L、Health Certificate 等）の基本知識

---

## 1. このシステムは何をするものか

本システムは、水産物の輸入案件ごとに発生する複数の貿易書類をAIが読み取り、以下を自動で行う**業務ワークベンチ**です。

- 書類の種別判定・バージョン管理（Draft / Final、修正版の履歴管理）
- 記載内容の抽出と正規化（単位・日付・表記ゆれの統一）
- **書類間の不一致検出**（重量・数量・Container No.・原産国など）
- **不足書類の検出**（必要書類チェックリストとの照合）
- 輸出者向け確認・修正依頼メールの**草案**作成
- 検疫所・税関向け提出準備資料の組み立てと出力
- 案件の進捗管理とすべての操作の監査記録

### 1.1 大原則: Human-in-the-loop

AIは**整理・確認・説明・草案作成**を行いますが、**判断はしません**。

| AIが行うこと | 人間（あなた）が行うこと |
|---|---|
| 書類の読み取り・値の抽出 | 抽出値の確認・修正 |
| 不一致・不足の検出 | 差異の受容 / エスカレーションの判断 |
| メール草案の作成 | 草案の承認と**実際の送信**（メールソフトで手動） |
| 提出資料の組み立て | 提出内容の最終確定と**実際の申請** |
| HS Code欄の準備 | HS Codeの決定 |

**本システムは外部へメールを送信しません。行政システム（NACCS等）にも接続しません。**
承認済みの草案・出力資料を、あなたが自分の手で送信・提出してください。

### 1.2 抽出値の「根拠」について

AIが抽出したすべての値には、以下が必ず付いています。

- **参照元書類**（どの書類から取ったか）
- **ページ番号**
- **根拠テキスト**（書類上の原文）
- **信頼度**（%表示。95%以上=緑 / 80%以上=黄 / それ未満=赤）

値をクリック（根拠欄を展開）すれば、いつでも「この値はどこから来たのか」を確認できます。
AIは書類に書かれていない値を**推測して埋めることはありません**。見つからない項目は空欄のままです。

## 2. 起動とログイン

1. システム管理者から案内されたURL（社内環境では通常 `http://localhost:3000`）をPCのブラウザで開きます。
2. 本MVP版にログイン機能はありません。開くとすぐにダッシュボードが表示されます。

> 💡 本システムはPC画面用に設計されています。複数書類の比較を行うため、なるべく大きな画面でご利用ください。

## 3. 画面構成

画面上部のヘッダーから、いつでも次の2つに移動できます。

| メニュー | 内容 |
|---|---|
| **ダッシュボード** | 全案件の一覧。緊急度・状態・問題件数を一目で確認 |
| **人間確認センター** | あなたの判断が必要な項目だけを優先度順に表示 |

### 3.1 Shipment Dashboard（案件一覧）

各案件について以下が表示されます。

- **緊急度** — 「緊急」（赤）/「通常」/「低」
- **案件ID・ルート** — 例: `VN-JP-001`（Vietnam → Japan）
- **商品・ETD/ETA**
- **状態** — 現在の工程（例: 書類受領中、人間確認待ち、検疫草案承認待ち）
- **書類** — 必要書類のうち受領済みの数（例: `3 / 5`）
- **問題** — 未解決の指摘件数（⚠付き）
- **承認待ち** — あなたの承認を待っている項目数

案件IDをクリックすると詳細画面へ移動します。

### 3.2 Shipment詳細画面（メイン作業画面）

3つのカラムで構成されています。

**左カラム: 書類**
- 必要書類チェックリスト（✅受領済み / ⬜未受領）
- 受領書類の一覧（種別・ファイル名・バージョン・Draft/Final・有効版/旧版）
- 書類アップロード欄

**中央カラム: 統一Shipmentデータ**
- 全書類から集約された項目一覧（商品名、重量、Container No.、金額…）
- 各値の正規化結果と原文
- 根拠欄（クリックで参照元・ページ・原文・他書類の値を表示）
- 各行の **[確認] [修正]** ボタン

**右カラム: Agentの指摘と次のアクション**
- 未解決の問題（重要度バッジ付き）と推奨アクション
- 各問題の **[差異を受容] [エスカレーション]** ボタン
- 状況に応じた次工程ボタン（メール草案作成、検疫準備へ進める など）
- 連絡文草案と **[承認] [却下]** ボタン

画面右上のリンクから **書類間比較** と **進捗・監査ログ** に移動できます。

### 3.3 書類間比較画面

すべての項目について、各書類の記載値を横並びで比較します。

| 判定 | 意味 |
|---|---|
| **一致**（緑） | 正規化後、全書類で同じ値（例: 「2026-07-18」と「July 18, 2026」は一致扱い） |
| **⚠ 警告**（赤背景） | 書類間で値が異なる。確認が必要 |
| **単一記載**（灰） | 1つの書類にしか記載がない（比較不能。不一致ではない） |
| **未記載** | その書類には記載なし |

### 3.4 進捗・監査ログ画面

- **ワークフロー進捗**: 10工程（書類受領 → 情報抽出 → … → 保存）のどこにいるか
- **Audit Log**: すべての状態変化・操作の履歴。「誰が（AI Agent / 担当者）・いつ・何を・なぜ」が時系列で記録され、後から監査できます

### 3.5 人間確認センター

全案件を横断して「あなたの判断待ち」だけを表示します。

- **高優先度** — 重大な不一致など、エスカレーション判断が必要なもの
- **通常** — 表記ゆれ確認、草案承認など
- **低リスク** — 軽微な確認
- **承認待ち** — 提出資料・メール草案などの承認リクエスト

ここから直接 [承認] [差異を受容] [エスカレーション] を実行できます。

## 4. 基本的な業務フロー（はじめから終わりまで）

### STEP 1: 書類をアップロードする

1. ダッシュボードから案件を開く
2. 左カラムの「書類アップロード」でPDFを選択（**複数まとめて選択可**、1ファイル20MBまで）
3. **[アップロードして抽出]** をクリック

システムが自動で行うこと:
- 書類種別の判定（Invoice / Packing List / B/L / Health Certificate / Certificate of Origin）
- ファイル名に「draft」等が含まれる場合は Draft 版として扱う
- 項目の抽出・正規化
- 必要書類チェックリストの更新
- 書類間の検証（不一致・不足・妥当性チェック）

処理が終わると、中央カラムに抽出結果、右カラムに指摘が表示されます。

### STEP 2: 抽出値を確認する

中央カラムで各値を確認します。

- 値が正しい → **[確認]** をクリック（「人間が確認済み」マークが付く）
- 値が違う → **[修正]** をクリックし、正しい値を入力
  - 修正すると自動で再検証が走り、関連する指摘が更新されます
  - 原文の値は履歴として保持されます

> 💡 全項目を確認する必要はありません。指摘に関係する項目と、提出資料に使う重要項目（商品名・重量・金額・Container No.等）を優先してください。

### STEP 3: Agentの指摘に対応する

右カラムの指摘には4種類あります。

| 種類 | 例 | 推奨対応 |
|---|---|---|
| **不一致** | Net WeightがInvoice 10,000 kg / Packing List 9,800 kg | 輸出者へ確認 → 修正版を依頼 |
| **不足書類** | Certificate of Origin未受領 | 輸出者へ発行状況を確認 |
| **記載なし** | Health CertificateにSeal No.なし | 追記可否を輸出者へ確認 |
| **妥当性警告** | ETAがETDより前 / 単価×数量≠合計金額 | 入力誤り・書類誤りを確認 |

重要度バッジの意味:
- **要エスカレーション**（赤） — 重大差異。原則、上長・専門担当者の判断を仰ぐ
- **要確認**（黄） — 人間の確認が必要
- **軽微**（緑） — 参考情報（根拠不足の通知など）

各指摘への操作:
- **[差異を受容]** — 問題ないと判断し、解消済み扱いにする（例: 表記ゆれだが同一商品と確認できた）
- **[エスカレーション]** — 専門担当者へ引き継ぐ。案件は「エスカレーション中」で停止し、対応完了後に **[エスカレーション対応完了]** で再開

### STEP 4: 輸出者へ確認・修正を依頼する

1. 右カラムの **[輸出者向け確認メール草案を作成]** をクリック
2. 未解決の指摘をまとめた英文メール草案が生成されます
3. 内容を確認し、**[承認]** または **[却下]**
4. 承認すると案件は「輸出者回答待ち」になります

> ⚠ **システムはメールを送信しません。** 承認した草案をコピーし、お使いのメールソフトから送信してください。

### STEP 5: 修正版書類を受け取ったら

輸出者から修正版（例: `Packing_List_..._revised.pdf`）が届いたら、**同じアップロード欄から**アップロードしてください。

- 自動で「修正版受領」となり、再抽出・再検証が走ります
- 新しい書類が **v2（有効版）**、元の書類は **旧版（履歴）** として保存されます
- 修正により解消した指摘は自動で「解消」になります

### STEP 6: 検疫・税関の提出準備へ進める

未解決の指摘がすべてなくなったら（解消・受容・エスカレーション対応済み）:

1. **[検疫提出準備へ進める]** をクリック
   - 未解決の問題が残っている場合は進めません（件数が表示されます）
2. 検疫所向け提出準備パッケージが生成され、「検疫草案承認待ち」になります
3. **検疫パッケージ（JSON / CSV）** をダウンロードして内容を確認
   - CSVはExcelでそのまま開けます（項目・値・参照元・根拠・信頼度・確認状況）
4. 問題なければ **[検疫提出資料を承認する]**
5. 続いて税関・通関業者向け資料が生成されます。同様に確認し **[税関申告資料を承認する]**

> ⚠ パッケージの **HS Code欄は常に「未確定」** です。HS Code・関税・法令適用の最終決定は、通関業者と相談のうえ人間が行ってください。実際の届出・申告も本システムの外で行います。

### STEP 7: 輸入許可後

- **[納品を手配する]** → コンテナ引取り・納品の手配を記録
- **[納品完了にする]** → 納品完了を記録
- **[案件を保存（アーカイブ）]** → 案件を確定保存。以後は履歴として参照のみ

すべての操作は Audit Log に残っているため、後からの監査・振り返りが可能です。

## 5. よくある質問（FAQ）

**Q. アップロードできるファイル形式は？**
A. PDF のみ（1ファイル20MBまで）。文字情報を持つPDFを推奨します。スキャン画像だけのPDF（文字情報なし）は本バージョンでは対象外です。

**Q. 書類種別が誤判定された / 「その他書類」になった**
A. ファイル名に書類種別が分かる語（invoice、packing、BL、health等）を含めて再アップロードしてください。誤った書類は旧版として履歴に残り、実務には影響しません。

**Q. 抽出された値が間違っている**
A. 中央カラムの **[修正]** で正しい値に直してください。修正は「人間が修正済み」として記録され、以後の検証・提出資料には修正後の値が使われます。原文も履歴として残ります。

**Q. 「不一致」と「根拠不足」の違いは？**
A. 「不一致」は複数の書類に**異なる値が書かれている**状態。「根拠不足」は**値が見つからず判定できない**状態です。根拠不足は誤りとは限りません。書類を確認し、必要なら値を補完してください。

**Q. 同じ種類の書類を間違えて2回アップロードした**
A. 新しい方が有効版（v2）になり、古い方は自動で旧版（履歴）になります。実害はありませんが、有効版が正しいものか左カラムで確認してください。

**Q. 「差異を受容」はどんなときに使う？**
A. 差異はあるが業務上問題ないと**あなたが判断した**とき。例: 「Frozen Shrimp」と「Frozen Vannamei Shrimp」が同一商品だと確認できた場合。受容の操作は記録に残ります。

**Q. エスカレーションすると何が起きる？**
A. 案件が「エスカレーション中」で停止し、通常の操作ボタンが隠れます。専門担当者の判断が済んだら **[エスカレーション対応完了]** で通常フローに戻ります。

**Q. メール草案の内容を少し直したい**
A. 本バージョンでは草案の直接編集はできません。**[却下]** して問題の状態を整えてから再生成するか、承認後にメールソフト側で修正して送信してください。

**Q. 過去の案件を見たい**
A. アーカイブ済み案件もダッシュボードに表示され、詳細・比較・監査ログを参照できます（操作はできません）。

## 6. 本システムが「行わないこと」（重要）

以下は仕様として意図的に行いません。必ず人間が実施してください。

- ❌ 外部へのメール自動送信
- ❌ 検疫所・税関・NACCS等への自動申請・接続
- ❌ HS Codeの最終決定、関税・法令適用の最終判断
- ❌ 問題の自動握りつぶし（すべての指摘は人間が処理するまで残ります）
- ❌ 書類に書かれていない値の推測・補完
- ❌ 根拠データ・監査ログの削除

## 7. 用語集

| 用語 | 意味 |
|---|---|
| Shipment / 案件 | 1回の輸入取引単位。複数の書類が紐づく |
| 有効版 / 旧版 | 同種書類のうち現在使用する版 / 履歴として保存される過去の版 |
| 正規化 | 単位・日付・表記ゆれを統一した値（例: 「10,000 kg」→「10000 kg」） |
| 根拠（Evidence） | 抽出値の出典となる書類上の原文とページ |
| 信頼度 | AIの抽出に対する確からしさ（%）。低い値は必ず人間が確認 |
| 指摘（Issue） | 検証で見つかった不一致・不足・警告 |
| エスカレーション | 専門担当者・上長へ判断を引き継ぐこと |
| Audit Log | すべての操作・状態変化の監査記録 |

## 8. 困ったときは

- 画面が更新されない → ブラウザを再読み込み（F5）
- アップロードがエラーになる → PDF形式か・20MB以下かを確認
- その他の不具合 → 発生時刻・案件ID・操作内容を添えてシステム管理者へ連絡してください（Audit Logから状況を再現できます）
